总结

  • go语言中的格式化动词
    %v：按照值的本来值输出。
    %+v：在 %v 基础上，对结构体字段名和值进行展开。
    %#v：输出 Go 语法格式的值，即输出值的 Go 语法表示形式。
    %T：输出值的类型。
    %%：输出一个百分号。
    %t：输出值的布尔表示（true 或 false）。
    %b：输出值的二进制表示。
    %o：输出值的八进制表示。
    %d：输出值的十进制表示。
    %x：输出值的小写十六进制表示。
    %X：输出值的大写十六进制表示。
    %U：输出值的 Unicode 编码。
    %q：输出值的单引号括起来的 Go 语法表示形式。
    %s：输出值的字符串表示形式。
    %p：输出指针的十六进制表示。
    %c：输出值的 Unicode 字符。
    %f：输出值的浮点数表示。
    %e：使用科学计数法输出值（小写字母 e）。
    %E：使用科学计数法输出值（大写字母 E）。
    %g：根据值的大小和精度选择 %e 或 %f 以输出值。
    %G：根据值的大小和精度选择 %E 或 %f 以输出值。

  • Go语言中的引用类型包括切片（slice）、映射（map）、通道（channel）和接口（interface）

  • 在其他编程语言中，函数执行完毕后，其内部的变量通常会被销毁，无法再被访问。但在支持闭包的编程语言中，函数可以继续访问和操作定义时的环境变量，即使函数在定义时的环境已经不存在。

闭包的特点包括：

  1. 函数嵌套：闭包是通过在函数内部定义函数来创建的，内部函数可以访问外部函数的变量。
  2. 保留状态：闭包可以“记住”定义时的环境，即使外部函数已经执行完毕，闭包仍然可以访问和操作定义时的变量。
  3. 可以作为值传递：闭包可以像其他值一样被传递，可以将闭包作为参数传递给其他函数，或者将闭包赋值给变量。

在上面的例子中，我们定义了一个 counter 函数，它返回一个内部函数 increment。increment 函数通过闭包的方式引用了外部函数 counter 中的变量 count。

每次调用 counter 函数时，都会创建一个新的 count 变量，并将其初始化为 0。然后，increment 函数被返回，并保持了对 count 变量的引用。

在 main 函数中，我们首先调用 counter 函数，将返回的 increment 函数赋值给变量 c。然后，通过多次调用 c()，我们可以观察到计数器每次递增并返回递增后的值。

9.关于类型转化，下面语法正确的是（）。C

A. type MyInt int var i int = 1 var j MyInt = i

B. type MyInt int var i int = 1 var j MyInt = (MyInt)i

C. type MyInt int var i int = 1 var j MyInt = MyInt(i)

D. type MyInt int var i int = 1 var j MyInt = i.(MyInt)

解析：在Go语言中，类型转化可以通过将值从一种类型转换为另一种类型来完成。在这个选项中，使用了类型转化操作符MyInt(i)将i从int类型转换为MyInt类型，然后将结果赋值给j。

其他选项中存在语法错误或不合法的类型转化操作：

  • 选项 A 中的赋值操作 var j MyInt = i 是一种隐式类型转换，试图将 int 类型的值直接赋给 MyInt 类型的变量 j。然而，Go 语言不支持不同基本类型之间的隐式类型转换。
  • 选项 B 中使用了显式的类型转换 (MyInt)i，但是在 Go 语言中，只有在接口类型与具体类型之间存在关联时才能进行显式类型转换。因此，将一个非接口类型（如 int）转换为自定义类型（如 MyInt）是不被允许的。
  • 选项 D 中使用了类型断言 i.(MyInt)，但是类型断言只适用于接口类型的值。由于 i 是一个 int 类型的变量，而不是接口类型，因此无法进行类型断言操作。

9.下面程序正确的输出结果是。D

A.1 2

B.2 3

C.3 2

D.1 3

这段代码定义了一个简单的链表结构 ListNode，每个节点包含一个整数值 Val 和一个指向下一个节点的指针 Next。

接下来，代码定义了一个名为 fn 的函数，它接受一个指向 ListNode 的指针作为参数。在该函数中，它将传入节点的值更新为下一个节点的值，并将当前节点的 Next 指针指向下一个节点的下一个节点，相当于将当前节点从链表中移除。

在 main 函数中，首先创建了一个链表的头节点 head，它的值为 1，下一个节点为值为 2 的节点，第二个节点的下一个节点为值为 3 的节点，最后一个节点的 Next 指针为 nil，表示链表的结束。

然后，通过调用 fn(head.Next) 将链表的第二个节点传递给函数 fn 进行处理。在函数内部，被传入的节点的值被更新为下一个节点的值（即将节点 2 的值更新为节点 3 的值），并且当前节点的 Next 指针指向了下一个节点的下一个节点（即节点 3 的下一个节点）。

最后，在 main 函数中，通过打印 head.Val 和 head.Next.Val，我们可以看到链表头节点的值为 1，而它的下一个节点的值已经变为 3，因为节点 2 被移除了。

因此，这段代码的执行结果将会输出 1 3。
